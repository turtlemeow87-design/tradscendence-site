# Bug Report — BR-006
**Site:** https://www.soundbeyondborders.com  
**Author:** Hunter Eastland  
**Date Filed:** February 25, 2026  
**Status:** Resolved  

---

## Summary
All pill remove buttons (×) across the instrument and genre selectors had no accessible name, making them unannounced and unusable for screen reader users.

---

## Environment
| Field | Details |
|---|---|
| URL | https://www.soundbeyondborders.com/contact |
| Browser | Chrome |
| OS | Windows |
| Tool | Chrome DevTools — Elements Panel, Accessibility Section |
| Standard | WCAG 2.1 AA |

---

## Element
```html
<button type="button" data-value="Other">×</button>
```

---

## Steps to Reproduce
1. Navigate to `/contact`
2. Open the Genre(s) dropdown and select any genre, including "Other"
3. Close the dropdown — pill(s) appear
4. Open DevTools → inspector cursor → click any × pill button
5. Observe Accessibility section — Name is empty

---

## Expected Result
Each × pill remove button should have an accessible name so screen readers can announce its purpose to keyboard and assistive technology users.

## Actual Result
Name field is empty in the accessibility tree for every pill button across both instrument and genre selectors. A screen reader user would reach these buttons with no announcement — no name, making it impossible to understand their purpose.

---

## Impact
**Low-Medium** — Affects screen reader users making any instrument or genre selection on the contact form. Every pill remove button on the form was affected, not just "Other".

---

## Fix Applied
Added a dynamic `aria-label` using a template literal in the `updatePills()` function in `contact.astro`. Since pills are generated dynamically by JavaScript, the fix needed to be applied in the script rather than static HTML:
```javascript
pill.innerHTML = `
  ${value}
  <button type="button" aria-label="Remove ${value}" data-value="${value}">&times;</button>
`;
```

This generates context-aware labels for every pill automatically — e.g. "Remove Surbahar", "Remove Classical", "Remove Other" — covering all instruments and genres in a single fix.

## Verification
Fix prototyped and confirmed working in Chrome DevTools — Accessibility panel updated to show Name populated after aria-label was applied. Verified on production after deployment — all pill remove buttons across instrument and genre selectors now announce correctly to screen readers.

**Low** — Narrow impact path but genuine WCAG accessibility gap. Fix is a one-line change with broad coverage.
